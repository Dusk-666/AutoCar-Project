# AutoCar-Project

基于 STM32F103C8T6 的智能循迹小车项目。当前版本使用 8 路红外循迹模块、舵机前轮转向、双路编码器反馈和电机闭环控制，在 `NORMAL / PASS / RECOVER` 三态状态机下实现稳定循迹、宽黑区通过和脱线恢复。

## 项目特性

- **8 路红外扫描**：通过 `AD0~AD2 + OUT` 轮询读取 1~8 号通道，获取比传统 4 路更细的线位信息
- **状态机循迹**：保留 `NORMAL / PASS / RECOVER` 三态结构，便于后续继续扩展十字路口、直角弯和指定转向
- **舵机渐进转向**：舵机目标角度按误差逐步逼近，不会瞬间大角度跳变
- **电机闭环控制**：编码器测速 + 双轮 PI 闭环，输出更稳定
- **速度渐变与保护**：目标速度采用斜坡过渡，并保留最小速度、最大速度和误差减速策略
- **串口调试输出**：支持打印 8 路传感器状态、误差、目标速度、反馈速度和 PWM 输出

## 硬件连接

### 1. 8 路红外循迹模块

当前工程适配的是带地址选择的 8 路数字循迹模块：

| 模块信号 | 单片机引脚 | 方向 | 说明 |
| :-- | :-- | :-- | :-- |
| `OUT` | `PA9` | 输入 | 当前被选中通道的数字输出 |
| `AD0` | `PA10` | 输出 | 地址位 0 |
| `AD1` | `PA11` | 输出 | 地址位 1 |
| `AD2` | `PA12` | 输出 | 地址位 2 |

地址选择与通道对应关系：

| `AD2` | `AD1` | `AD0` | 通道 |
| :-- | :-- | :-- | :-- |
| 0 | 0 | 0 | CH1 |
| 0 | 0 | 1 | CH2 |
| 0 | 1 | 0 | CH3 |
| 0 | 1 | 1 | CH4 |
| 1 | 0 | 0 | CH5 |
| 1 | 0 | 1 | CH6 |
| 1 | 1 | 0 | CH7 |
| 1 | 1 | 1 | CH8 |

通道从左到右依次为 `1 ~ 8`。当前代码按以下逻辑解释传感器输出：

- **检测到黑线**：模块输出低电平 `0`
- **检测到白底**：模块输出高电平 `1`

## 控制逻辑简介

### 1. 8 路传感器采样

控制周期内，程序会依次切换 `AD0~AD2`，轮询 8 个通道，并生成：

- `channel[8]`：8 路原始命中状态
- `mask`：8 位压缩状态
- `active_count`：当前命中黑线的通道数量

### 2. 位置误差计算

当前循迹误差按左右对称的多级权重计算，通道越靠外，误差权重越大；误差越大，说明小车偏离中心越明显。

### 3. 三态状态机

- `NORMAL`：常规循迹，按误差计算速度差并更新舵机目标
- `PASS`：进入宽黑区或临时异常区时，保持方向通过
- `RECOVER`：重新见线后短暂恢复，再切回 `NORMAL`

### 4. 舵机控制

舵机不会瞬间打满角，而是：

1. 根据当前位置误差计算目标脉宽
2. 限幅在允许转角范围内
3. 每个控制周期只向目标逼近一小步

这样可以避免机械冲击，也更利于过弯时保持平顺。

### 5. 电机闭环

- 上层循迹模块输出左右目标速度
- `Motor` 模块对目标做斜坡处理
- 编码器提供实时反馈
- 左右轮分别用 PI 闭环输出 PWM

## 参数位置

主要参数集中在以下文件：

- `Core/Inc/Track.h`：循迹控制周期、传感器接口、误差权重、舵机参数、状态机参数
- `Core/Inc/Motor.h`：电机 PI、目标速度限制、速度斜坡、直线补偿
- `Core/Inc/Encoder.h`：编码器滤波和方向配置

## 调试建议

### 1. 先确认传感器顺序

查看串口里的 `sen=xxxxxxxx` 输出，确认：

- 左到右是否真的是 `1 ~ 8`
- 压到黑线上时是否输出 `1`
- `mask` 与实际通道变化是否一致

### 2. 再调舵机

优先调：

- 舵机中心脉宽
- 最大偏转量
- 每周期转动步长

### 3. 最后调速度环

优先观察：

- 左右轮反馈速度是否接近
- 大误差时是否降速足够
- 过弯时是否有明显过冲或抖动
